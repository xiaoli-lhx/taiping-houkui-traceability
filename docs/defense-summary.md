# 毕业答辩系统说明

## 1. 当前系统已实现功能清单

### 1.1 用户与角色

- 用户登录
- 用户注册
- 注册状态查询
- 管理员审核注册申请
- 获取当前用户信息
- 前后端结合的角色区分与专属门户
- 已支持角色：
  - 管理员 `admin`
  - 茶农 `farmer`
  - 企业 `enterprise`
  - 监管方 `regulator`
  - 消费者 `consumer`

### 1.2 溯源管理

- 茶叶批次创建与查询
- 按批次查看完整阶段记录
- 新增溯源阶段记录
- 可按批次码、溯源码、产品编号进行查询

### 1.3 品质辅助评估

- 录入外形、色泽、香气、滋味四项指标
- 自动计算综合得分
- 自动生成等级判定
- 展示指标明细
- 展示品质评估雷达图

### 1.4 监管审核

- 监管方查看批次审核状态
- 监管方查看审核历史记录
- 监管方提交审核记录
- 审核提交后，批次审核状态同步变化

### 1.5 数据统计分析

- 统计概览指标
- 产量分布图
- 品质等级占比图
- 品质指标趋势图

### 1.6 消费者公开查询

- 消费者通过溯源码、批次码或产品编号查询公开溯源信息
- 查看公开批次信息、阶段链路和最新品质等级
- 支持真实二维码展示和扫码跳转公开查询链接

### 1.7 消费者中心

- 消费者登录后查看查询历史
- 收藏感兴趣的批次
- 提交反馈内容

## 2. 演示流程建议

建议按“注册 -> 管理员审核 -> 企业录入 -> 监管审核 -> 消费者查询”的顺序演示。

### 第一步：注册与管理员审核

- 用茶农、企业或消费者角色提交注册
- 用 `admin_demo / 123456` 登录管理员门户
- 在注册审核页通过申请

### 第二步：企业端登录并创建批次

- 使用 `enterprise_demo / 123456`
- 登录时选择角色 `enterprise`
- 进入企业门户批次管理页
- 创建一个新的太平猴魁批次

### 第二步：补录阶段记录

- 进入批次详情页
- 新增加工、包装或流通阶段记录
- 展示完整溯源链路

### 第三步：进行品质评估

- 进入品质评估页
- 录入外形、色泽、香气、滋味分值
- 展示综合得分、等级和雷达图

### 第四步：展示统计分析

- 进入统计分析页
- 展示产量分布、等级占比、指标趋势
- 说明这些结果可以为后续茶叶质量管控和经营决策提供参考

### 第五步：监管方审核

- 使用 `regulator_demo / 123456`
- 登录时选择角色 `regulator`
- 进入监管审核页
- 查看批次审核状态
- 提交审核意见并展示审核结果变化

### 第六步：消费者查询

- 打开公开查询页
- 输入前面演示批次的溯源码
- 展示公开溯源信息和品质结果
- 如需演示账号化能力，可再用消费者账号登录查看查询历史与收藏

## 3. 角色说明

### 企业

- 创建与维护茶叶批次
- 录入加工、包装、流通等信息
- 录入品质评估
- 查看统计分析

### 茶农

- 查看相关批次信息
- 在批次详情页补录阶段记录
- 不显示企业侧的新建批次入口

### 监管方

- 查看统计分析
- 进入监管审核页
- 查看并提交审核记录

### 消费者

- 可匿名公开查询
- 登录后进入消费者中心
- 查看查询历史、收藏和反馈

### 管理员

- 审核注册申请
- 查看用户列表
- 管理账号状态

## 4. 图表与可视化说明

本系统当前前端采用 `React + Ant Design` 管理系统界面，并使用 `@ant-design/plots` 实现图表展示。系统按角色拆分专属门户，并保留匿名公开查询入口。

### 品质评估雷达图

- 展示四个核心品质指标
- 可直观说明不同批次的品质差异
- 对应开题报告中的“品质辅助评估模块”

### 产量分布图

- 按产地展示批次总重量
- 反映产量结构

### 品质等级占比图

- 展示不同等级评估记录数量
- 可用于答辩中说明品质整体情况

### 品质指标趋势图

- 按日展示指标变化趋势
- 体现系统具备基础统计与分析能力

### 公开查询二维码

- 页面展示真实二维码
- 二维码内容为公开查询链接，并自动带入溯源码参数
- 更贴近茶叶包装扫码溯源的实际场景
- 支持通过前端环境变量配置局域网访问地址，便于手机扫码演示
