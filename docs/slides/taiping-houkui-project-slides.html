Project Overview

太平猴魁茶叶溯源与品质辅助评估系统

一个面向毕业设计讲解场景的前后端联调项目，围绕茶叶批次溯源、品质辅助评估、监管审核和消费者公开查询形成完整业务闭环。

Go + Gin + GORM + MySQL React + Vite + TypeScript RBAC + 图表 + 二维码查询

重点不是单点功能，而是把“多角色协同 + 溯源链路 + 质量评价 + 对外查询”连成一条可演示主线。

本套讲解稿默认采用专业科技风，适合答辩展示、项目讲解和演示录屏，也为后续补充系统截图预留了位置。

02 / 07

Why This System

项目背景与目标

茶叶产业中，批次信息、生产阶段、品质结果和监管动作往往分散，消费者查询链路也不够直观。本项目尝试用一个统一系统把这些信息组织起来。

讲解重点：既要说明“为什么做”，也要说明“做成了什么闭环”，而不是只罗列接口或页面。
溯源可见

按批次码、溯源码、产品编号组织批次与阶段记录，形成链式展示。

品质可解释

围绕外形、色泽、香气、滋味四项指标自动算分并判定等级。

监管可追踪

监管方可查看批次审核状态、审核历史并提交审核意见。

消费可查询

消费者可匿名公开查询，也可登录后查看历史、收藏与反馈。

04 / 07

Core Walkthrough

核心业务主线

这套系统最适合按“注册审核 -> 企业建批次 -> 阶段记录 -> 品质评估 -> 监管审核 -> 消费查询”的顺序讲解，因为它能完整体现多角色协同。

1
注册与审核

茶农、企业或消费者注册后进入待审核状态，由管理员在注册审核页通过申请。

2
企业创建批次

企业账号进入批次管理页，创建新的太平猴魁批次，形成后续溯源和评估主体。

3
补录阶段记录

在批次详情页补录种植、采摘、加工、包装或流通等阶段信息，形成完整链路。

4
品质辅助评估

录入四项指标，系统自动计算综合得分、等级结论，并展示雷达图结果。

5
监管审核

监管方查看批次审核状态和历史记录，提交审核动作后同步改变审核结果。

6
消费者公开查询

通过二维码或溯源码查询公开信息，登录后还可记录历史、收藏与提交反馈。

05 / 07

Modules

功能模块拆解

前端路由和页面已经按角色门户区分，当前实现的重点是“不同角色看到不同入口，但围绕同一个批次对象协同工作”。

  • 管理员门户：注册审核、用户管理。
  • 企业门户：批次管理、品质评估、统计分析。
  • 监管门户：批次审核、统计查看。
  • 消费者模块：公开查询、收藏、反馈、查询历史。
角色门户与权限体验

通过前端路由和 RoleGuard 控制角色可见菜单，配合后端 JWT + RBAC 形成双层约束。

品质评估与图表结果

系统支持录入四项指标，自动输出综合得分、等级判定和品质雷达图。

统计分析与经营参考

提供概览指标、产量分布、等级占比和指标趋势图，支撑质量管控与经营决策说明。

公开查询与消费者中心

支持二维码跳转公开查询页，并在登录后记录历史、收藏感兴趣批次和提交反馈。
